כדי להציג את הקלף שזכית בו בתוך ה-Activity בצורה יפה, אנחנו צריכים להגדיר מיכל (Container) ב-XML שבו הקלף יופיע. כרגע ה-XML שלך מכיל רק את התיבה והכפתורים.

הכי נכון יהיה להוסיף LinearLayout ריק בין התיבה לבין הכפתורים, ולשם להזריק את הקלף ברגע שהאנימציה מסתיימת.

1. עדכון ה-XML (activity_loot_box.xml)

2. עדכון קוד ה-Java ב-LootBoxActivity
עכשיו נשתמש במתודה UIUtils.createViewCard בתוך ה-Handler. שים לב שאני מוסיף cardContainer.removeAllViews() כדי שאם המשתמש יפתח שוב, הקלף הקודם ייעלם והחדש יופיע.

Java

// בתוך ה-onCreate של LootBoxActivity
LinearLayout cardContainer = findViewById(R.id.cardContainer);
AnimatedChestView chestView = findViewById(R.id.chestView);
Button btnOpen = findViewById(R.id.btnOpen);

btnOpen.setOnClickListener(v -> {
    // איפוס המיכל לפני פתיחה חדשה
    cardContainer.removeAllViews();
    
    // הפעלת אנימציית התיבה
    chestView.startAnimation();

    // המתנה לסיום האנימציה (800ms)
    new android.os.Handler().postDelayed(() -> {
        
        // 1. הגרלת הקלף מהדאטהבייס
        Card wonCard = dbHelper.getRandomCard();
        
        if (wonCard != null) {
            // 2. הוספה לאוסף ב-SQL
            dbHelper.addCardsToCollection(wonCard.getId(), 1);
            
            // 3. יצירת ה-View של הקלף באמצעות ה-Utils שלך
            View cardView = UIUtils.createViewCard(getLayoutInflater(), cardContainer, wonCard, this);
            
            // 4. הזרקת ה-View לתוך ה-Layout במסך
            cardContainer.addView(cardView);
            
            // בונוס: אנימציה קטנה של הופעה (Fade In) לקלף
            cardView.setAlpha(0f);
            cardView.animate().alpha(1f).setDuration(500).start();
        }
    }, 800); 
});
מה קורה כאן?
cardContainer: זה ה-LinearLayout שמשמש כ"עוגן" לקלף.

removeAllViews(): מונע מצב שבו קלפים נערמים אחד על השני אם לוחצים על פתיחה כמה פעמים.

UIUtils.createViewCard: הקוד משתמש בדיוק במתודה שסיפקת. היא מנפחת (Inflate) את ה-XML של הקלף ומחזירה View מוכן.

addView(cardView): השלב הסופי שמציג פיזית את הקלף על המסך של המשתמש.
